S01

디자인 원칙

PRINCIPLES

통합기획서 v1 [docs/통합기획서.html:1942](통합기획서.html#L1942) "디자인 원칙 6가지"를 그대로 채택하되, 본 문서에서 프론트·CMS 환경별로 적용 강도를 조정한다.

원칙 1
따뜻한 중립
차가운 흰색·순검정 대신 크림·먹빛. 프론트는 크림 100%, CMS는 크림 70% + 흰 카드로 작업대 인상.
원칙 2
타이포 주연
세리프(제목)+산세리프(본문)+모노(숫자). 프론트는 세리프 적극 사용, CMS는 세리프 절제(테이블 시인성 우선).
원칙 3
여백은 사치가 아니다
섹션 96px·카드 내부 28px. 프론트 그대로 채택. CMS는 정보 밀도 우선 → 카드 내부 20px로 축소.
원칙 4
코랄 한 점
CTA·KPI·경고만 코랄. 한 화면에 코랄 3개 이상이면 디자인 회귀. 프론트·CMS 동일.
원칙 5
모션은 0.2초 이하
0.15~0.2s. 의사결정 속도 유지. 프론트·CMS 동일.
원칙 6
숫자는 큰 모노
KPI·금액은 JetBrains Mono. 0/O, 1/l 혼동 방지. 프론트 결과 화면·CMS 대시보드 KPI 적용.

프론트 vs CMS 캐릭터 차이

프론트 (공개 빌더)
  • 인상: 따뜻한 도구. 엑셀 대체.
  • 배경: claude-cream (#F0EEE6) 우세.
  • 여백: 넉넉(96px 섹션).
  • 세리프: 적극 사용 (Hero·결과).
  • 그림자: 부드러운 shadow-sm.
  • 모서리: 12~16px 큰 둥글기.
  • 버튼: 큰 패딩, 코랄 Primary 1개만.
CMS (관리자 작업대)
  • 인상: 단단한 작업대. 정보 밀도.
  • 배경: claude-ivory (#FAF9F5) + 흰 카드.
  • 여백: 절제(섹션 48px).
  • 세리프: 헤더만. 본문·표는 산세리프.
  • 그림자: 더 옅게 (shadow-sm만).
  • 모서리: 8~10px 작은 둥글기.
  • 버튼: 컴팩트, 다양한 보조 색.

S04

공통 컴포넌트 카탈로그

COMPONENTS

shadcn/ui를 base로 하되, Claude-inspired 토큰으로 재스킨. 모든 컴포넌트는 데스크톱·모바일 양쪽에서 동일한 시각·동작.

① Button

엑셀 다운로드 취소 자세히 보기 삭제
variant용도한 화면 최대
primary (코랄)주요 CTA. 화면당 1개1
secondary (흰 + 보더)보조 액션제한 없음
ghost (투명)인라인 액션·취소제한 없음
danger (빨강)삭제·되돌릴 수 없음1
icon-only편집 ✎ 삭제 ✕ 추가 +리스트 행당 2~3

크기: sm 32px (CMS 테이블 행) · md 40px (기본) · lg 48px (프론트 Hero CTA). 텍스트 좌우 패딩은 높이의 0.5배.

② Input · Number Spinner

  • 높이: 40px (기본) · 32px (CMS 테이블 인라인) · 48px (프론트 큰 입력).
  • 모서리: 10px.
  • focus: 2px outline (--teal-500), border-color 동시 변경.
  • 숫자 입력은 ± 스피너 버튼 항상 함께 (모바일 친화).
  • 금액 입력은 자동 ₩ 접두 + 천 단위 콤마 표시.

③ Select / Combobox / Search

  • 관광지·식당 선택은 검색형 Combobox (10개 이상 옵션 시).
  • 차량·가이드 등급은 RadioGroup (4개 이하).
  • 시즌은 RadioCardGroup — 시각적 카드 4개 가로 배치.

④ Tag / Chip

활성 시즌 적용 비활성 완료 초안

⑤ Table

  • CMS 전용. 행 높이 48px·헤더 56px.
  • thead는 teal-900 배경 흰 텍스트.
  • tbody 행 hover 시 ink-50.
  • 액션 컬럼은 우측 고정·아이콘 버튼.
  • 20행 이상 시 페이지네이션 또는 가상 스크롤.

⑥ Card

  • 프론트: 흰 배경 · 12px radius · shadow-sm · top border 3px (코랄/teal/gold 강조).
  • CMS: 흰 배경 · 8px radius · 보더만(그림자 없음).

⑦ Modal · Sheet

  • 데스크톱: 중앙 정렬 모달. 너비 480px(small) / 720px(large).
  • 모바일: 하단 시트 (Sheet from bottom). 드래그 핸들 노출.
  • 배경 dim: rgba(0,0,0,0.4) · backdrop-blur(8px).
  • ESC·바깥 클릭으로 닫힘 (Danger 모달은 ESC만 가능, 외부 클릭 비활성).

⑧ Toast

  • 우측 상단(데스크톱) · 상단(모바일) 슬라이드 인.
  • 4초 자동 사라짐. Danger는 사용자 확인까지 유지.
  • variant: success(초록) / info(파랑) / warn(노랑) / error(빨강).

⑨ Tabs · Accordion

  • 탭: 빌더 좌측 사이드바(인원·일정·시즌·마진·환율).
  • 아코디언: FAQ(P04)·모바일 사이드바 시트 안.

⑩ Slider

  • What-if 시뮬레이션의 인원 슬라이더 · 환율 조정.
  • thumb: 코랄 원 16px · track: ink-200 4px · active track: 코랄.

⑪ Badge·Avatar·Tooltip·Skeleton·Progress·Stepper

  • Badge: 작은 점 + 숫자 (DAU·신규 N개).
  • Avatar: 관리자 로그인 이름 이니셜 (대표 "P").
  • Tooltip: 200ms 지연 후 노출. 키보드 focus 시에도 표시.
  • Skeleton: shimmer 1.5s · 이미지·테이블 로딩 시.
  • Progress: Day별 완성도 (선택 항목 0/10).
  • Stepper: 미사용 (Phase 0은 단일 화면).
S05

상태·피드백 패턴

STATES

모든 데이터 호출 화면은 5가지 상태를 명시적으로 디자인한다. "loading 깜빡임" "empty 빈 페이지" "error 무반응" 회피.

IDLE (기본)
초기 진입. localStorage에 작업 없음. CTA "새 패키지 만들기" 중앙 강조.
LOADING
마스터 데이터 fetch. Skeleton (행 5개) 200ms 이후만 표시 (깜빡임 방지).
SUCCESS
저장·삭제·계산 완료. Toast 2~4초. UI는 즉시 갱신.
ERROR

② 계산식 — 두 입력이 만나는 곳

// STEP A — 사용자 마진 적용 (모든 채널 공통)
const netPrice    = tierCost × (1 + margin.net);         // 100 × 1.12 = 112
const retailPrice = netPrice  × (1 + margin.retail);      // 112 × 1.35 = 151.2
const salePrice   = retailPrice × (1 + vatRate);          // 151.2 × 1.10 = 166.32

// STEP B — 채널별 수수료 차감 (채널마다 다른 결과)
for (const channel of channels) {
  const channelFee  = salePrice × channel.commission_rate;  // 도매 10%면 16.63
  const cardFee     = salePrice × channel.card_fee_rate;    // 온라인 2.8%면 4.66
  const netRevenue  = salePrice − channelFee − cardFee;     // 실수령액
  const realProfit  = netRevenue − tierCost;                // 실제 이익
  const realProfitRate = realProfit / salePrice × 100;     // ★ 채널별 순이익률
  channels[i].realProfitRate = realProfitRate;
}
